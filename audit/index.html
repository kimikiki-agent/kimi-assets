3. 專案關係圖

通訊 資料 排程 前端 阻塞 衝突 ● 節點顏色 = 健康狀態（🟢🟡🔴）

⚠️ 衝突

AvatarPlan ↔ AgentCore — 架构边界与拆分方向
agent-core-split.md 计划将 AgentCore 与 AvatarPlan 完全解偶为独立可部署模块，但当前 AvatarPlan 前端已直接指向 AgentCore:8200 并依赖其全权限 session 模式。拆分计划与现状耦合矛盾，需明确边界：是保持紧耦合还是推进完全拆分
AgentChat ↔ AvatarPlan — 对话 session 与上下文管理
AgentChat 通过 Telegram 建立独立的对话记忆与 session 管理，AvatarPlan 通过桌面应用有单独的 session。两条对话路径的上下文如何同步？是否应共用 AgentCore 的统一 session 管理？当前无明确规范

⏳ 阻塞

AvatarPlan — Tauri 桌面封裝 [todo] → 等待 AvatarController — EmotionMapper.ts + PriorityMixer.ts [todo]
Avatar 完整的情绪响应系统需要这两个组件实现，否则无法开始桌面应用的可视化集成
ContentMarketing — remotion-影片产线 [todo] → 等待 RiveGen — Phase 2/3（Animation Timeline）无排期
要在 Threads 发布高质量影片内容，需要 RiveGen 完成 Animation 支持。当前 RiveGen 仅完成静态图形，未排期動畫功能
AgentCore — user_model 实现 [gap] → 等待 AvatarPlan — React 整合 [in_progress]
user_model 需要采集用户交互数据（对话打分、行为反馈），AvatarPlan Chat UI 完成后才有采集端点

🔗 依賴

AgentCore → AvatarPlan
user_model 实现需要 AvatarPlan 提供用户交互反馈采集接口（对话打分、行为记录等），才能完成「即时个人化」的短路径学习
AvatarPlan → AvatarController
Tauri 桌面封裝 [todo] 依赖 AvatarController 的两个未完成组件（EmotionMapper.ts、PriorityMixer.ts），当前阻塞完整集成
ContentMarketing → RiveGen
remotion-影片产线 [todo] 需要 RiveGen Phase 2/3（Animation Timeline）支持，才能驱动 Rive 动画渲染为影片
MorningBrief → NotionTaskRunner
晨间自动修复机制（inbox_buildup remediation）依赖 NotionTaskRunner 的任务执行能力，目前直接调用 research-inbox 但尚无通用任务调度接口
