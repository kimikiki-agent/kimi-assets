⏳ 阻塞

AvatarController (EmotionMapper.ts 與 PriorityMixer.ts 實作) → 等待 RiveGen (Phase 2/3 Animation + State Machine)
AvatarController checklist [todo] EmotionMapper 和 PriorityMixer 需要 .riv 檔案原生支援 State Machine 與 Animation。RiveGen 當前停留在 Phase 1B（驗證工具 + 基礎 Shape），Phase 2/3 尚未排入 checklist，導致 AvatarController 無法推進。
AvatarPlan (Broken 欄修復) → 等待 AgentCore (Claude CLI session 正常化)
AvatarPlan 對話與檔案操作依賴 AgentCore 的 CLI session，但 CLI 從 /tmp 執行產檔、subprocess 中 open 命令失效等問題未解。此問題需在 AvatarPlan 正式發布前完全清除。
AgentCore (Handoff 萃取品質調優 [todo]) → 等待 MorningBrief (daily_reflection + detector pipeline 完整性)
AgentCore self-learning.md 的「長路徑」由 MorningBrief daily_reflection 實現。AgentCore checklist [todo] 的「Handoff 萃取品質調優」需要 MorningBrief 的完整 detector 管線（inbox / audit / content / sessions / jobs / checklist）來驗證長期學習效果。MorningBrief v2 已完成，可開始協同驗證。

AvatarPlan → AgentCore
AvatarPlan 前端（Tauri a5 + React）指向 AgentCore:8200 WebSocket API。AvatarPlan broken 欄提及的 Claude CLI /tmp 路徑問題直接影響 AgentCore 後端的檔案操作可靠性，兩者需要聯動修復。

AvatarController EmotionMapper.ts 與 PriorityMixer.ts [todo] 實作需要 RiveGen 提供成熟的 State Machine 與 Animation 支援（Phase 3）。RiveGen Phase 2/3 缺 checklist 導致 AvatarController 無法確定依賴完成時間。
ContentMarketing → AvatarPlan
ContentMarketing 計畫文件 2026-04-11-remotion-影片產線.md 提到升級社群內容為「Rive 角色動畫影片」。此計畫依賴 AvatarPlan 的 Avatar 動畫資源與表情庫輸出，需要協調資源共用機制。
