3. 專案關係圖

通訊 資料 排程 前端 阻塞 衝突 ● 節點顏色 = 健康狀態（🟢🟡🔴）

⚠️ 衝突

AgentCore ↔ AgentChat — Session 單例性（single session mode）
AgentCore goals 定義「單 session 模式：同時只有一個活躍對話，避免 context 衝突」。但 AgentChat goals 要求「Telegram 雙向自然語言對話」，若 Telegram 有活躍對話，AvatarPlan 或 CLI 是否能同時對話？goals.md 中未明確定義 session 並行約束的具體邊界
AvatarController ↔ AvatarPlan — Avatar 控制路徑（control pathway）
avatar-controller-integration.md 明確指出「目前有三套獨立的 avatar 控制路徑同時存在，彼此未協調：路徑 A (AgentAvatar.tsx)、路徑 B (AvatarController)、路徑 C (State Machine)」。三條路徑的職責邊界未定義，導致設計衝突與潛在功能重疊
AvatarPlan ↔ AvatarController — 情緒 → 參數轉換責任邊界
avatar-controller-integration.md 指出「EmotionMapper 與 State Machine 的情緒→參數轉換責任邊界未定義，導致三套控制路徑共存且互相衝突」。需明確 EmotionMapper、State Machine、AgentAvatar.tsx 的責任分工，避免邏輯重疊

⏳ 阻塞

AgentChat (超長對話記憶驗收) → 等待 AgentCore (Handoff 萃取品質調優 [todo])
AgentChat 要求「對話記憶維持最近 N 則上下文，跨訊息語意連貫」，但 AgentCore 的 Handoff 萃取品質優化仍待完成。記憶精準度未驗證前，AgentChat 無法通過完整的長對話驗收
AvatarPlan (Avatar 表情系統完整交付) → 等待 AvatarController (EmotionMapper + PriorityMixer [todo])
AvatarPlan goals 要求「Avatar 表情系統：State Machine 支援情緒連動」。AvatarController 的 EmotionMapper 和 PriorityMixer 尚在 [todo]，導致表情—情緒映射功能無法完成
AvatarPlan (Chat UI 流暢體驗完成) → 等待 AvatarPlan (Broken: /tmp file I/O 與 open 指令失效)
AvatarPlan broken 記錄「Claude CLI 從 cwd=/tmp 執行，檔案在 /private/tmp 使用者找不到；open 指令在 subprocess 中不運作」。此問題直接阻礙檔案操作與視窗開啟功能，計畫待桌面版 (Tauri a5) 完成後修復
RiveGen Phase 2 實作 → 等待 RiveGen Phase 1 (schema 管理完整性確認)
RiveGen Phase 2 擴充多 Shape 類型需更新 schema（目前 360 個 JSON defs）。Phase 1 的 schema 自動生成與版本管理策略未明確文件化，Phase 2 無法推進
ResearchBot (所有實作) → 等待 ResearchBot (goals.md 與 plans 缺失)
ResearchBot 為完全 stub 狀態，goals.md 未定義、plans 缺失。無基線定義，無法開始任何實作工作，也無法被 Autoresearch 評估，形成專案進度完全堵塞
ContentMarketing (庫存 ≤3 篇目標達成) → 等待 ContentMarketing (自動補貨機制驗證缺失)
Goals 要求庫存 ≤3 篇自動補貨穩定運作，但目前庫存 7 篇。checklist 無補貨機制驗證項，無法確認系統是否定期執行，目標達成缺乏具體行動方案

🔗 依賴

AgentCore → AgentChat
AgentChat goals 要求「對話記憶維持最近 N 則上下文，跨訊息語意連貫」。此功能依賴 AgentCore 的 Chat Handoff 能力，而 AgentCore checklist 中「Handoff 萃取品質調優」仍為 [todo]，導致 AgentChat 記憶精準度風險
AvatarController → AvatarPlan
AvatarPlan goals 成功標準要求「Avatar 表情系統：State Machine 支援情緒連動」，由 AvatarController 的 EmotionMapper 實現。EmotionMapper 與 PriorityMixer 在 checklist 中標記 [todo]，會延遲 AvatarPlan 完整表情系統的交付
RiveGen → AvatarController
AvatarController 需要透過 RiveGen 輸出 .riv 動畫檔案。若 RiveGen Phase 2（多 Shape、漸層）未完成，會限制 Avatar 動畫的複雜度與視覺層次，影響使用者體驗
Autoresearch → 所有專案
Autoresearch Project Auditor 定期掃描所有專案 goals.md vs checklist 對齊度。ResearchBot 為 stub（goals.md 未定義），導致 Autoresearch 無法正確評估該專案，可能遺漏告警或產生誤判
NotionTaskRunner → ApprovalBridge
NotionTaskRunner 預設自動執行任務，高風險任務由 ApprovalBridge 審核。兩系統的 risk-tiers 定義需對齊，否則任務風險分級不一致，可能導致應審核的任務被自動執行，或應自動執行的任務被卡在審核
AgentPulse → MorningBrief
MorningBrief 依賴 AgentPulse 的異常偵測結果（6 個 detector）進行反省與自動調整。若 AgentPulse 的異常覆蓋率不足，MorningBrief 無法發現所有問題，自動調整的決策品質受影響
SessionHandoff → 所有專案
Goals 提及的效能優化（START 並行讀、END Smart 合併）會影響 session 轉換速度。若未實現，跨 session turn 數過多會降低所有專案的交班流暢度與使用者體驗
