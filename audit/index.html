AvatarPlan 內 avatar-controller-integration.md 指出「目前有三套獨立的 avatar 控制路徑同時存在」（路徑 A — AgentAvatar.tsx、路徑 B — EmotionMapper、路徑 C），但無明確統一方案。AvatarController checklist 中 EmotionMapper.ts [todo] 和 PriorityMixer.ts [todo] 應該是解決方案，但對應關係未在 AvatarPlan 計畫中明確。建議在 avatar-controller-integration.md 中補充解決方案細節與實作清單。

⏳ 阻塞

AvatarPlan 的 agent-workspace-template 初始化與複製驗證 → 等待 AgentToolkit 的 agent-init 實作（[todo]）
AvatarPlan 計畫 agent-workspace-template.md 提到「新 agent workspace 應按照 template 初始化」，但依賴 AgentToolkit 完成 agent-init 和 skill-sync 機制。建議在 AgentToolkit checklist 中明確 agent-init 的依賴優先級。
ContentMarketing 的 Remotion 影片產線驗證 → 等待 RiveGen Phase 2 完成（多 Shape、漸層、Path）+ AvatarPlan 的角色素材確定
Remotion 影片產線計畫中提到需要「Rive 角色動畫完整資產」作為素材。目前 RiveGen Phase 1B 已完成（validator + rigger），但 Phase 2 未排期。建議在 RiveGen checklist 中補充 Phase 2 計畫與截止日期。

AgentToolkit 的 agent-init（todo）需完成，才能支援 AvatarPlan 的 agent-workspace-template 初始化流程。新 agent workspace 的 CLAUDE.md 和 plans/ 結構需依照 template 標準化。

RiveGen Phase 2（多 Shape、漸層、Path）完成後，AvatarController 的參數化介面規格（avatar-parameter-spec.md）才能在實際 .riv 上驗證。ContentMarketing 的 Remotion 影片產線需要 Rive 完整動畫資產作為素材。
