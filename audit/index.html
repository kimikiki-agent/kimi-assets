3. 專案關係圖

通訊 資料 排程 前端 阻塞 衝突 ● 節點顏色 = 健康狀態（🟢🟡🔴）

⏳ 阻塞

Autoresearch (skill-production-monitoring) → 等待 所有 skills（session-handoff、content-digest、research-inbox 等）穩定運作並被生產環境真實使用
skill-production-monitoring 依賴「真實使用觀察」而非模擬測試，需等待所有 skills 在實戰中驗證穩定性後，才能建立可靠的品質基準線（baseline）。
RiveGen (Phase 2 Shape & Gradient) → 等待 AvatarController (EmotionMapper.ts + PriorityMixer.ts) 完成 Phase 1
RiveGen 新增的 Shape 和漸層參數規格需與 AvatarController 的參數混合邏輯協調，避免實作時出現不兼容。當前 AvatarController 的兩項 todo 會定義出最終參數集。

🔗 依賴

AgentToolkit → SessionHandoff
AgentToolkit 計畫建立 AI/shared/skills/ + skill-manifest.json 與 skill-sync 機制，SessionHandoff 的多專案技能同步功能需依賴此設施更新 END 時的知識匯出邏輯。
AgentCore (user_model) → AvatarPlan
AgentCore self-learning.md 規劃的 user_model 短路徑需 Chat UI（AvatarPlan）採集對話互動數據（提問、修正、參考內容偏好），才能訓練個人化模型。API 界面與數據收集策略需協調。
RiveGen (Phase 2/3) → AvatarController
RiveGen Phase 3 的 Animation + State Machine 支援新增後，AvatarController 的 EmotionMapper 和 PriorityMixer 需擴展參數集以利用新能力。目前 AvatarController 的 PriorityMixer 仍在 todo，整合時序需協調。
Autoresearch (skill-production-monitoring) → AgentComm
skill-production-monitoring 需 AgentComm 或其他 I/O 層支援自動記錄 skill 呼叫紀錄（入參、出參、耗時），方可累積品質數據。當前 AgentComm 無此紀錄機制。
